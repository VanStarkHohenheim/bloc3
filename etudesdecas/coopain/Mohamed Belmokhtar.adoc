=== Question 1.2 — Provenance des informations
**Votre réponse :** Tableau listant grossièrement les sources ("Combinaison de champs", "Calcul sur date").

.Remarque - Manque de précision
====
Votre analyse est beaucoup trop superficielle pour une épreuve de BTS. Il fallait identifier précisément les attributs sources (ex: `Collecte.date`, `Taureau.numeroIE`) et détailler les transformations techniques (ex: concaténation au format 128 CRT, calcul de l'écart en jours depuis 2002).

=== Questions 1.3 et 1.4 — Requêtes SQL (Tarifs et Stocks)
**Votre réponse :** Requêtes avec `JOIN` et `LEFT JOIN`.

[NOTE]
.Évaluation - Validé
====
Les syntaxes SQL sont exactes.
Pour la question 1.4, l'utilisation du `LEFT JOIN` pour le calcul du stock total est une excellente pratique. Cela permet de conserver dans l'affichage les taureaux dont le stock est tombé à zéro, contrairement à un `INNER JOIN` classique. À reproduire en Edc BTS.
====

=== Question 1.5 — Modèle relationnel
**Votre réponse :** Liste textuelle des entités (Vache, DemandeInsemination, Inseminateur, Secteur, etc.) avec une description vague de leurs liens.

[WARNING]
.Erreur - Formalisme manquant
====
Une simple liste n'est pas un schéma relationnel. Il faut impérativement détailler les attributs et formaliser les contraintes d'intégrité (souligner les clés primaires, utiliser le symbole `#` pour les clés étrangères) afin de lier techniquement la demande à l'adhérent, à la vache et au lot.
====
